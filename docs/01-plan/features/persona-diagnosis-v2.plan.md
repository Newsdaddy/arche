## 1. 개요

### 1.1 목적
기존 간단한 "콘텐츠 유형 테스트"를 **심층 소셜 페르소나 진단**으로 업그레이드.
사용자의 성격, 능력, 직업, 서비스/제품, ICP를 체계적으로 분석하여 맞춤형 콘텐츠 전략을 제공한다.

### 1.2 핵심 변경사항
| 항목 | 기존 (v1) | 신규 (v2) |
|-----|----------|----------|
| 히어로 섹션 | "나의 콘텐츠 유형은?" | "나의 소셜 페르소나는?" |
| CTA | "무료 테스트 시작하기" | "진단 시작하기" |
| 질문 수 | 3문항 (30초) | 25-30문항 (10-15분) |
| 분석 프레임워크 | 단순 유형 분류 | 5개 프레임워크 통합 |
| 결과물 | 유형명 + 한 줄 팁 | 종합 리포트 + 콘텐츠 전략 |

## 2. 적용 프레임워크 (5개)

### 2.1 SWOT 분석 (기본)
- **목적**: 자기분석의 골격
- **구성**:
  - S (강점): 능력, 경험, 성격 강점
  - W (약점): 부족한 스킬, 제약사항 → 약점도 콘텐츠가 된다
  - O (기회): 트렌드, 플랫폼, 타겟 고객
  - T (위협): 경쟁자, 지속 가능성 리스크
- **SWOT Mix 전략**:
  - S×O: 강점으로 기회 공략
  - W×O: 약점을 기회로 전환 (성장 스토리)
  - S×T: 강점으로 위협 방어
  - W×T: 약점+위협 회피 전략

### 2.2 Hero's Journey (영웅 서사 5막)
- **목적**: 스토리 기반 정체성 수집
- **구성**:
  1. **평범한 일상**: 현재 직업, 일상, 과거 경험
  2. **시련과 고난**: 겪은 실패, 어려움, 약점
  3. **깨달음과 전환**: 터닝포인트, 배운 교훈
  4. **변화와 성장**: 현재 진행 중인 도전
  5. **비전과 소명**: 하고 싶은 일, 전하고 싶은 메시지
- **콘텐츠 연결**: 각 막이 콘텐츠 소재가 됨

### 2.3 Enneagram (에니어그램)
- **목적**: 성격/동기 분석 (MBTI 대체)
- **9가지 유형**:
  | 유형 | 이름 | 핵심 동기 | 핵심 두려움 |
  |-----|-----|---------|----------|
  | 1 | 개혁가 | 올바름 | 부패/결함 |
  | 2 | 조력가 | 사랑받음 | 사랑받지 못함 |
  | 3 | 성취가 | 가치 인정 | 무가치함 |
  | 4 | 개인주의자 | 고유함 | 평범함 |
  | 5 | 탐구자 | 이해/지식 | 무능력 |
  | 6 | 충성가 | 안전/지지 | 불안정 |
  | 7 | 열정가 | 자유/행복 | 고통/제한 |
  | 8 | 도전가 | 통제/힘 | 취약함 |
  | 9 | 평화주의자 | 평화/조화 | 갈등/분리 |
- **콘텐츠 연결**: "왜 이 이야기를 해야 하는가" 설명

### 2.4 Value Proposition Canvas + ICP
- **목적**: 비즈니스/서비스/제품 분석, 타겟 고객 정의
- **ICP (Ideal Customer Profile)**:
  - 인구통계: 나이, 직업, 위치, 소득
  - 심리: 고통점, 욕망, 두려움
  - 행동: 사용 플랫폼, 콘텐츠 소비 패턴
- **가치 제안 (Value Proposition)**:
  - 고객의 Job to be Done
  - 해결하는 Pain
  - 제공하는 Gain

### 2.5 능력 교차점 (Ikigai 변형)
- **목적**: 차별화 포인트 도출
- **4가지 원**:
  - 잘하는 것 (Skill)
  - 좋아하는 것 (Passion)
  - 세상이 필요로 하는 것 (Demand)
  - 돈이 되는 것 (Monetization)
- **유니크 포지션**: 2-3가지 능력의 교차점
- **예시**: "마케팅 + 심리학 + 글쓰기" → "설득의 심리학 콘텐츠"

### 4.2 아키타입 정의 (6가지)

| 아키타입 | 이모지 | 설명 | Enneagram 연관 |
|---------|-------|-----|---------------|
| 회복의 전사 | ⚔️ | 실패를 극복하고 다시 일어선 스토리 | 8, 3 |
| 늦깎이 도전자 | 🌱 | 늦게 시작했지만 열정으로 성장 중 | 7, 1 |
| 지식 수집가 | 📚 | 깊은 전문성으로 인사이트 전달 | 5, 1 |
| 공감의 연결자 | 🤝 | 사람들의 마음을 연결하는 다리 | 2, 9 |
| 창조적 실험가 | 🎨 | 새로운 시도와 독특한 관점 | 4, 7 |
| 실용의 안내자 | 🧭 | 검증된 방법으로 길을 안내 | 6, 1 |

## 5. 기술 구현 계획

### 5.1 파일 구조
```
app/
├── diagnosis/
│   ├── page.tsx          # 진단 시작 (선택: 간이 vs 심층)
│   ├── quick/
│   │   └── page.tsx      # 간이 테스트 (기존 3문항)
│   ├── deep/
│   │   └── page.tsx      # 심층 진단 (29문항)
│   └── result/
│       └── page.tsx      # 결과 페이지 (통합)
│
lib/
├── diagnosis/
│   ├── questions.ts      # 질문 데이터
│   ├── analyzer.ts       # 분석 로직 (SWOT, Enneagram 등)
│   ├── archetypes.ts     # 아키타입 정의
│   └── report.ts         # 리포트 생성
│
types/
└── diagnosis.ts          # 진단 관련 타입
```

### 5.2 데이터 저장
- `persona_results` 테이블 활용 (이미 생성됨)
- 저장 항목:
  - `diagnosis_type`: 'quick' | 'deep'
  - `archetype`: 아키타입 코드
  - `strengths`, `weaknesses`, `opportunities`, `threats`: SWOT
  - `skills`: 능력 교차점 배열
  - `unique_position`: 유니크 포지션 문장
  - `content_style`: 추천 콘텐츠 스타일
  - `raw_answers`: 원본 답변 (JSON)

### 5.3 AI 연동
- 결과 분석에 Groq API 활용
- 프롬프트: 답변 → SWOT/아키타입/콘텐츠 전략 도출
